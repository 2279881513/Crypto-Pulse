# CryptoPulse 回测策略指标说明

> 最后更新: 2026-07-14 | 版本: v3.1

> 基于**价格行为/结构分析**（道氏理论趋势结构 + 支撑阻力突破 + 成交量确认 + 追踪止损）的 BTC 永续合约量化策略

---

## 一、策略核心思想

**不做预测，只做跟随**。完全基于价格行为：

| 要素 | 说明 |
|------|------|
| **趋势结构** | 道氏理论：更高的高点和低点 = 上升，更低的高点和低点 = 下降 |
| **支撑阻力** | 用 pivot 自动识别关键价位 |
| **成交量确认** | 放量 > 1.8×MA20，且 K 线收盘在极端位 |
| **挂单突破** | 入场价 = pivot 价位（模拟 stop 单），不等收盘确认 |
| **追踪止损** | 价格到达 trail_activate 后止损移至成本价 |

---

## 二、入场信号

### Pivot 检测

- **LOOKBACK** = 30 根 K 线（用于趋势判断和关键价位识别）
- **left=3, right=3**：pivot high = 左右各 3 根范围内最高，pivot low = 左右各 3 根范围内最低
- 不足 2 个 pivot 时趋势判定为盘整（不产生突破信号）

### 入场条件

| 信号 | 做多 | 做空 | 信心 |
|------|------|------|:----:|
| **放量突破/破位** | 上升趋势 + 收盘破 pivot high + 多方 K 线 + 放量>1.8× | 下降趋势 + 收盘破 pivot low + 空方 K 线 + 放量>1.8× | 70~75 |
| **连续实体** | 连续 3 根阳线 + 放量>1.8× + 多方 K 线 | 连续 3 根阴线 + 放量>1.8× + 空方 K 线 | 60 |

> 放量 > 2.0× 时信心提升至 75（突破信号）或维持 60（连续实体）。

### K 线位置过滤

| 方向 | 条件 |
|:----:|------|
| **多方破位** | 收盘 > 开盘 **且** 实体 > 影线（多方主导） |
| **空方破位** | 收盘 < 开盘 **且** 实体 > 影线（空方主导） |

### 连续实体附加条件

连续实体入场需要前两根 K 线的实体均 > 平均价格的 **0.2%**，防止影线/十字星被误判为实体延续。

### 信号优先级

```
1. 放量突破/破位（高信心）
2. 连续实体延续（中等信心）
```

同一根 K 线上两个信号互斥，优先尝试突破信号。

## 三、止盈止损

### 参数定义

```
ATR = 14 周期平均真实波幅（用于动态调整止盈止损距离）
波幅 = 最近阻力位 − 最近支撑位（用于大波动行情自适应）
```

### 止损

```
ATR × 1.5（固定倍数，不可调）
做多：入场价 − 1.5 × ATR
做空：入场价 + 1.5 × ATR
```

### 止盈

```
TP 距离 = max(波幅, ATR × 2.0)    ← 确保 TP/SL ≥ 1.33
TP1 = 入场价 ± TP 距离 × 1.0      ← 实际用作 trail_activate（追踪止损触发位）
TP2 = 入场价 ± TP 距离 × 1.5
TP3 = 入场价 ± TP 距离 × 2.0
```

> **注意**：API 回测中，TP1 被拉远到 `tp_dist × 4`（约 8×ATR），trail_activate 保留在 `tp_dist × 1.0`。这是为了让追踪止损先于固定止盈触发，避免被 TP1 提前截断。

### 追踪止损

```
到达 trail_activate（≈ tp_dist × 1.0）后：
  → 止损移至入场价（保本）
  → 之后止损跟随价格：做多 = max(当前止损, 最高价 - ATR×2.0)
                         做空 = min(当前止损, 最低价 + ATR×1.5)
```

### 出场逻辑

| 结果 | 条件 | 说明 |
|:----:|------|------|
| ✅ **止盈** | K 线最高/低价触及止盈价 | 固定止盈离场 |
| ❌ **止损** | K 线最高/低价触及止损价 | 固定止损或追踪止损离场 |
| 🎯 **追踪止盈** | 止损触发但盈利 | SL 触发时价格已在成本之上，归类为追踪止盈 |
| ⏱️ **超时** | 验证期内未触发以上两者 | 用验证期结束时的收盘价判断盈亏 |

## 四、风控

| 功能 | 阈值 | 说明 |
|------|:----:|------|
| **保本检查** | TP1 利润 ≥ **0.10%** | 双边手续费阈值，不足则放弃并标记 💰 |
| **止损冷却** | 默认 **5 分钟** | 止损触发后暂停交易 N 分钟（可开关可调） |
| **连续亏损保护** | 同方向 **3 次** | 连续亏损后暂停该方向交易，盈利后双方重置 |

保本阈值计算：
```
fee_rate = 0.05%（默认，可在 UI 调节）
min_tp1_pct = fee_rate × 2 × 100 = 0.10%
```

## 五、策略执行流程

```
每根 K 线
  → 取 30 根识别 pivot（left=3, right=3）
  → 趋势结构判断（道氏理论：更高高/低点→上升，更低高/低点→下降）
  → 入场信号检查
      ├─ 放量破位（趋势 + 突破 pivot + 放量>1.8× + K 线位置过滤）
      └─ 连续实体（3 根同向 + 实体>0.2% + 放量>1.8× + K 线位置过滤）
  → 风控检查
      ├─ 止损冷却（上次止损后 N 分钟内不开仓）
      ├─ 连续亏损保护（同方向连亏 3 次暂停）
      └─ 保本检查（TP1 利润 ≥ 0.10%）
  → 执行交易
      ├─ 入场价 = pivot 价位（突破信号）或收盘价（连续实体）
      ├─ 止损 = ATR × 1.5
      ├─ trail_activate = tp_dist × 1.0
      └─ 止盈 = tp_dist × 4 / ×6 / ×8
  → 验证期（lookahead 根 K 线内监控）
      ├─ 触及止盈 → ✅ 止盈
      ├─ 触及止损 → ❌ 止损
      ├─ 到达 trail_activate → 止损移至成本价，后续追踪
      └─ 超时 → 收盘价判断盈亏
```

## 六、信号计算位置

策略实现在 `cryptopulse/api/app.py` 的 `_compute_signal()` 函数中（约 143~288 行），回测循环在同文件的 `api_backtest()` 路由中。

---

## 七、回测参数

| 参数 | 默认值 | 说明 |
|------|:------:|------|
| 验证值（lookahead） | 10 根 K 线 | 入场后检查多少根 K 线来判断盈亏 |
| 交易风格 | short_term (1m) | short_term=1m, medium_term=4H |
| 默认时间范围 | 最近 7 天 | 可通过 UI 或 API 参数调整 |
| 手续费 | 0.05% | 双边各 0.05%，可在 UI 仓位面板调节 |
| 本金 | 1000 USDT | 仅用于计算净利/ROI，不影响信号生成 |
| 杠杆 | 1× | 可在 UI 仓位面板调节 |
| 保本过滤 | 可选 | 启用后过滤利润 < 0.10% 的交易 |
| 止损冷却 | 5 分钟 | 可开关、可调节冷却时长 |
