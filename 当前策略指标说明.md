# CryptoPulse 回测策略指标说明
> 最后更新: 2026-07-13

> 基于**价格行为/结构分析**（道氏理论趋势结构 + 支撑阻力突破 + 成交量确认）的 BTC 永续合约量化策略

---

## 一、策略核心思想

**不做预测，只做跟随**。不依赖任何滞后技术指标（EMA、RSI、MACD、布林带、ADX等），完全基于：

| 要素 | 说明 |
|------|------|
| **趋势结构** | 道氏理论：更高的高点和低点 = 上升趋势，更低的高点和低点 = 下降趋势 |
| **支撑阻力** | 用摆动高低点（pivot）自动识别关键支撑/阻力位 |
| **成交量确认** | 所有信号必须放量（>20期均量的 1.5 倍，强力信号需 > 2 倍） |
| **突破/回踩** | 价格放量突破关键位 或 回踩关键位失败时入场 |

---

## 二、入场信号

### 趋势判断（Pivot检测）

取最近 **30根K线**，识别摆动高低点（pivot），左右各 **2根K线**确认（left=2）：

```
pivot high = 某K线的最高价是左右各2根K线范围内的最高值
pivot low  = 某K线的最低价是左右各2根K线范围内的最低值
```

| 趋势方向 | 条件 |
|---------|------|
| **上升趋势** | 最近2个 pivot high 递增 AND 最近2个 pivot low 递增 |
| **下降趋势** | 最近2个 pivot high 递减 AND 最近2个 pivot low 递减 |
| **盘整** | 其他情况 |

### 入场条件（按优先级）

| 优先级 | 信号类型 | 做多条件 | 做空条件 | 信心值 |
|:------:|---------|---------|---------|:-----:|
| 1 | **放量破位** | 上升趋势 + 收盘突破最近 pivot high + 放量>1.5× | 下降趋势 + 收盘跌破最近 pivot low + 放量>1.5× | 70~75 |
| 2 | **回踩确认** | 最低价触及支撑位但收盘反弹 | 最高价触及阻力位但收盘回落 | 65 |
| 3 | **连续实体** | 连续2根大阳线 + 放量 | 连续2根大阴线 + 放量 | 60 |

**强力确认**：成交量 > MA20 的 2 倍时，信心额外 +5。

### 放量确认

```
成交量比 = 当前K线成交量 / 20期成交量均值
成交量比 > 1.5 → 放量确认（入场基础条件）
成交量比 > 2.0 → 强力确认（信心提升）
```

## 三、观望（不进场条件）

| # | 条件 | 说明 |
|---|------|------|
| 1 | **K线不足30根** | 需要至少30根历史K线来识别结构 |
| 2 | **放量不足** | 成交量比 ≤ 1.5，无确认信号 |
| 3 | **无结构** | 盘整趋势（pivot高点和低点方向不一致）|
| 4 | **同方向冷却** | 同方向连续信号间隔 < 2根K线时跳过 |
| 5 | **止损冷却** | 止损触发后暂停交易5分钟，期间不开新仓 |
| 6 | **TP1利润不足** | TP1预期 < 双边手续费（默认0.10%）|
| 7 | **连续亏损保护** | 同方向连续亏损3次后暂停该方向交易 |

## 四、止盈止损

### 止损（固定 ATR 倍数）
```
止损 = ATR × 1.5
做多：入场价 − 1.5 × ATR
做空：入场价 + 1.5 × ATR
```

### 止盈（波浪扩展测量，优先）

止盈优先使用波浪扩展目标位，基于最近一个完整的 pivot 波幅：

```
波幅 = 最近 pivot high - 最近 pivot low

做空止盈：
  TP1 = 最近 pivot low  − 波幅 × 1.0（1:1等长扩展）
  TP2 = 最近 pivot low  − 波幅 × 1.618（黄金扩展）
  TP3 = 最近 pivot low  − 波幅 × 2.0（2倍扩展）

做多止盈：
  TP1 = 最近 pivot high + 波幅 × 1.0
  TP2 = 最近 pivot high + 波幅 × 1.618
  TP3 = 最近 pivot high + 波幅 × 2.0
```

**回测仅使用 TP1 出场**。

### 波浪测量示例（对应示例数据）

从 9:01 到 9:10 的上涨波：
- 低点 pivot：8002 → 8030 → 8057（低点持续抬高）
- 高点 pivot：8024 → 8048 → 8075（高点持续抬高）

从 9:11 开始的下跌波：
- pivot high: 8075 → 8065（高点降低 ✓）
- pivot low:  8057 → 8048（低点降低 ✓）
- 波幅 = 8075 − 8048 = 27点
- TP1 做空 = 8048 − 27 = **8021**（对应示例中 9:17 目标 8023）
- TP2 = 8048 − 27 × 1.618 = 8048 − 43.7 = **8004**
- TP3 = 8048 − 27 × 2.0 = **7994**

## 五、风控系统

### 止损冷却
- 止损触发后暂停交易 **5分钟**
- 冷却期内所有新开仓信号被阻止（橙色箭头标记 ⛔）

### 保本检查
- 进场前 TP1 利润 > 双边手续费（默认 0.10%），否则放弃

### 连续亏损保护
- 同方向连续亏损 **3 次** → 暂停该方向交易
- 任意方向盈利时，双方计数归零

### 风控显示
| 元素 | 说明 |
|------|------|
| 🟠 橙色箭头 ⛔ | 被风控阻止的信号 |
| ⚠️ 风控标签页 | 风控阻止统计、原因分布（可点击高亮） |

## 六、策略执行流程

```
每根K线
    │
    ├─→ 取最近30根K线，识别 pivot high / pivot low（left=2）
    │
    ├─→ 趋势结构判断
    │       ├─ pivot高点和低点均上升 → 上升趋势
    │       ├─ pivot高点和低点均下降 → 下降趋势
    │       └─ 其他 → 盘整（无入场信号）
    │
    ├─→ 入场信号检查（按优先级）
    │       ├─ 放量破位：趋势确认 + 突破/跌破 pivot + 放量>1.5×
    │       ├─ 回踩确认：触碰 pivot 但收盘折返
    │       └─ 连续实体：连续2根大实体同向K线
    │
    ├─→ 冷却检查 → 连续亏损检查 → 止损冷却检查 → 保本检查
    │
    └─→ 执行交易
            ├─ 止损: ATR × 1.5
            └─ 止盈: 波浪扩展 TP1（优先）| ATR × 2.0（fallback）
```

## 七、策略参数速查

| 参数 | 值 | 说明 |
|------|-----|------|
| LOOKBACK | 30根K线 | 结构识别的回溯长度 |
| pivot_left/right | 2根K线 | 确认摆动点的左右窗口 |
| vol_ratio_threshold | 1.5× | 放量确认阈值（>20期均量的1.5倍） |
| vol_strong_threshold | 2.0× | 强力放量确认阈值 |
| risk_n（fallback） | 2.0 | ATR止盈倍数（无波浪测量时用） |
| sl_mult | 1.5 | 止损倍数，止损 = 入场价 ± sl_mult × ATR |
| cool_bars | 2 | 同方向交易冷却（根K线） |
| sl_cooldown | 300s | 止损后冷却时间（5分钟） |
| max_consecutive_losses | 3 | 连续亏损保护阈值 |
| 默认费率 | 0.05% | 单边手续费 |
| 保本阈值 | 0.10% | 双边手续费 |
