# CryptoPulse 回测策略指标说明
> 最后更新: 2026-07-13 | 版本: v2 (Deepseek 优化版)

> 基于**价格行为/结构分析**（道氏理论趋势结构 + 支撑阻力突破 + 成交量确认 + 追踪止损）的 BTC 永续合约量化策略

---

## 一、策略核心思想

**不做预测，只做跟随**。完全基于价格行为：

| 要素 | 说明 |
|------|------|
| **趋势结构** | 道氏理论：更高的高点和低点 = 上升，更低的高点和低点 = 下降 |
| **支撑阻力** | 用 pivot 自动识别关键价位 |
| **成交量确认** | 放量 > 1.5×MA20，且 K 线收盘在极端位 |
| **挂单突破** | 入场价 = pivot 价位（模拟 stop 单），不等收盘确认 |
| **追踪止损** | 价格到达 TP1 后止损移至成本价 |

---

## 二、入场信号

### Pivot 检测
- **LOOKBACK** = 30 根K线
- **left=3, right=3**——减少噪声（v2 改进）
- pivot high = 左右各3根范围内最高
- pivot low  = 左右各3根范围内最低

### 入场条件（按优先级）

| 优先级 | 信号 | 做多 | 做空 | 信心 |
|:------:|------|------|------|:----:|
| 1 | **放量破位** | 上升趋势 + 收盘破 pivot high + 多方K线 + 放量 | 下降趋势 + 收盘破 pivot low + 空方K线 + 放量 | 70~75 |
| 2 | **回踩确认** | 低点触支撑但收盘反弹（阳线） | 高点触阻力但收盘回落（阴线） | 65 |
| 3 | **连续实体** | 连续3根阳线 + 放量 + 多方K线 | 连续3根阴线 + 放量 + 空方K线 | 60 |

**K线位置过滤（v2 新增）**：
- 多方破位：收盘 > 开盘 且 实体 > 影线（即阳线占K线主体部分）
- 空方破位：收盘 < 开盘 且 实体 > 影线（即阴线占K线主体部分）

## 三、止盈止损

### 止损
```
ATR × 1.5（固定，统一单位）
做多：入场价 − 1.5 × ATR
做空：入场价 + 1.5 × ATR
```

### 止盈（v2 改进：解决盈亏比缺陷）
```
TP距离 = max(波幅, ATR × 2.0)      ← 确保 TP/SL 比例 ≥ 1.33
TP1 = 入场价 ± TP距离 × 1.0
TP2 = 入场价 ± TP距离 × 1.5
TP3 = 入场价 ± TP距离 × 2.0
```

**波幅** = 最近 pivot high − pivot low

### 追踪止损（v2 新增）
```
当价格到达 trail_activate（≈ 入场价 ± TP距离×1.0）时：
  止损 → 入场价（成本价保本）
```

## 四、观望（不进场条件）

| # | 条件 | 说明 |
|---|------|------|
| 1 | **K线不足30根** | 无法识别结构 |
| 2 | **放量不足** | 成交量比 ≤ 1.5 |
| 3 | **趋势盘整** | pivot方向不一致 |
| 4 | **K线位置不佳** | 破位K线收盘位置不极端 |
| 5 | **同方向冷却** | 间隔 < 2根 |
| 6 | **止损冷却** | 触发后暂停5分钟 |
| 7 | **TP1利润不足** | 不够覆盖手续费（0.10%） |
| 8 | **连续亏损保护** | 同方向连续亏3次后暂停 |

## 五、改进记录（vs v1）

| # | Deepseek指出的问题 | v2 修复 |
|---|-------------------|---------|
| 1 | 盈亏比缺陷：止盈波幅 < 止损ATR | TP距离 = max(波幅, ATR×2.0)，确保 TP/SL ≥ 1.33 |
| 2 | 入场太迟：价格突破时已穿过TP1 | 入场价 = pivot价位（模拟 stop 挂单），不等收盘 |
| 3 | pivot left=2 过于敏感 | left=3，减少假突破 |
| 4 | 成交量确认不充分 | 增加 K 线收盘位置过滤（实体 > 影线）+ 连续3根确认 |
| 5 | 固定止损被插针扫损 | 增加追踪止损保护（保本后跟随） |
| 6 | 缺少让利润奔跑 | TP1 到达后止损移至成本价 |
