# CryptoPulse 回测策略指标说明
> 最后更新: 2026-07-11 00:45:00

> 基于多技术指标加权评分卡 + 周期共振 + ADX 趋势过滤的 BTC 永续合约量化策略

---

## 一、技术指标总览

| 指标 | 参数 | 用途 |
|------|------|------|
| **EMA** | 5 / 13 / 21 | 趋势排列：多头排列（短 > 中 > 长），空头排列（短 < 中 < 长） |
| **MACD** | 快线5 慢线13 信号线5 | 金叉/死叉 + 柱线伸长方向 |
| **RSI** | 周期14 | 超买 ≥ 70（看空），超卖 ≤ 30（看多） |
| **布林带（BB）** | 周期20, 标准差2.5 | 价格触碰上轨（超买）或下轨（超卖） |
| **ATR** | 周期14 | 波动率，用于止盈止损距离计算 |
| **ADX** | 周期14 | 趋势强度：≥25 有趋势，≥35 强趋势，<25 盘整 |
| **OBV** | 14周期EMA差 | 资金流向验证 |
| **VWAP** | 20周期 | 价格偏离度 |
| **成交量比** | 20周期均量 | 放量/缩量确认 |
| **微观结构** | OBI / 主动成交比 / 大单 | 辅助验证（权重仅5%） |

---

## 二、做多（LONG）— 入场条件

### 硬性准入条件（三者必须同时满足）

| # | 条件 | 说明 |
|---|------|------|
| 1 | **RSI ≤ 30** | 超卖区 |
| 2 | **价格 ≤ 布林下轨 + (中轨 − 下轨) × 0.3** | 价格靠近下轨 |
| 3 | **ADX ≥ 10** | 至少存在微弱趋势 |

### 加权评分卡（总分 > 阈值 且方向为 BULLISH → 做多）

| 评分项 | 权重 | 条件 |
|--------|:----:|------|
| EMA 排列 | 25% | 5 > 13 > 21 多头排列 |
| MACD | 10% | 零上金叉，柱线伸长 |
| RSI | 10% | 超卖区域值 |
| 动量 | 15% | 向上 |
| 布林带 | 10% | 触碰下轨 |
| OBV | 12% | 向上 |
| VWAP | 5% | 价格在 VWAP 上方 |
| 成交量比 | 8% | 放量 |
| 微观结构 | 5% | 正评分 |

> **ADX ≥ 35** → 总分 × 1.15（放大信号），且评分阈值从 ±30 降至 **±15**
>
> **ADX < 25** → 需总分绝对值 ≥ 35 才能入场（过滤盘整期假信号）
>
> **ADX ≥ 50 + 布林下轨触边** → 绕过评分阈值，**直接做多**（强趋势豁免）
>
> **ADX ≥ 50 + 布林上轨触边** → 绕过评分阈值，**直接做空**（强趋势豁免）

## 三、做空（SHORT）— 入场条件

### 硬性准入条件（三者必须同时满足）

| # | 条件 | 说明 |
|---|------|------|
| 1 | **RSI ≥ 70** | 超买区 |
| 2 | **价格 ≥ 布林上轨 − (上轨 − 中轨) × 0.3** | 价格靠近上轨 |
| 3 | **ADX ≥ 10，且 (ADX ≤ 45 或 ADX > 50)** | ADX∈(45,50] 时不做空（历史胜率仅33%），但ADX>50+BB上轨时强趋势豁免做空 |

### 加权评分卡（总分 < 阈值 且方向为 BEARISH → 做空）

与做多使用同一套 9 项指标，评分取负值，对称判断：

| 评分项 | 权重 | 条件 |
|--------|:----:|------|
| EMA 排列 | 25% | 5 < 13 < 21 空头排列 |
| MACD | 10% | 零下死叉，柱线缩短 |
| RSI | 10% | 超买区域值 |
| 动量 | 15% | 向下 |
| 布林带 | 10% | 触碰上轨 |
| OBV | 12% | 向下 |
| VWAP | 5% | 价格在 VWAP 下方 |
| 成交量比 | 8% | 放量 |
| 微观结构 | 5% | 负评分 |

## 四、观望（NEUTRAL / HOLD）— 不进场条件

| # | 条件 | 说明 |
|---|------|------|
| 1 | **评分绝对值 ≤ 阈值** | 正常阈值30，强趋势(ADX≥35)时阈值降至15 |
| 2 | **ADX < 25 且总分绝对值 < 35** | 盘整期过滤，强制中性 |
| 3 | **置信度 < 35** | 信号置信度不足 |
| 4 | **5m 趋势与 1m 信号冲突** | 周期共振不一致（周期冲突保护） |
| 5 | **TP1 涨幅不足以覆盖手续费** | 预期利润 < 交易成本（`min_tp1_pct`） |
| 6 | **进场冷却期** | 同方向连续信号间隔 < 2根K线时跳过；ADX>50+BB触边时豁免（间隔≥1根K线可进场） |
| 7 | **强趋势方向保护** | ADX>45 且方向与EMA趋势相反时强制观望（不做逆势） |
| 8 | **极端趋势保护** | ADX>80 强制观望，极端单边行情不开新仓 |

## 五、止盈（Take-Profit）— 出场规则

### 短线（1m / 5m 级别）— 三级止盈

| 层级 | 仓位比例 | 多头止盈价 | 空头止盈价 |
|:----:|:--------:|-----------|-----------|
| TP1 | 30% | 入场价 + ATR × 1.3 | 入场价 − ATR × 1.3 |
| TP2 | 30% | 入场价 + ATR × 2.2 | 入场价 − ATR × 2.2 |
| TP3 | 40% | 入场价 + ATR × 3.5（转为移动止损） | 入场价 − ATR × 3.5（转为移动止损） |

### 中线（4H / 1D 级别）— 三级止盈

| 层级 | 多头止盈价 | 空头止盈价 |
|:----:|-----------|-----------|
| TP1 | 入场价 + ATR × 2.0 | 入场价 − ATR × 2.0 |
| TP2 | 入场价 + ATR × 4.0 | 入场价 − ATR × 4.0 |
| TP3 | 入场价 + ATR × 7.0 | 入场价 − ATR × 7.0 |

### API 回测默认参数

- **TP_mult = 5.0**（使用单一止盈价，非三级止盈）

### 回测出场判断

| 方向 | 止盈条件 | 超时处理 |
|:---:|---------|---------|
| 做多 | 盘中最高价 ≥ 止盈价 → 止盈出场 | 最多480根K线（≈8小时），按收盘价判定盈亏 |
| 做空 | 盘中最低价 ≤ 止盈价 → 止盈出场 | 同上 |

## 六、止损（Stop-Loss）— 出场规则

| 级别 | 止损距离 | 多头止损价 | 空头止损价 |
|:----:|:--------:|-----------|-----------|
| 短线（signal.py） | ATR × 1.2 | 入场价 − ATR × 1.2 | 入场价 + ATR × 1.2 |
| 中线（signal.py） | ATR × 1.5 | 入场价 − ATR × 1.5 | 入场价 + ATR × 1.5 |
| API 回测默认 | ATR × 2.0 | 入场价 − ATR × 2.0 | 入场价 + ATR × 2.0 |
| Engine 实盘版（结合支撑位） | min(最近摆动低点下方, ATR × 2.0) | — | — |

### 回测出场判断

| 方向 | 止损条件 |
|:---:|---------|
| 做多 | 盘中最低价 ≤ 止损价 → 止损出场 |
| 做空 | 盘中最高价 ≥ 止损价 → 止损出场 |

## 八、策略执行流程

```
每根K线
    │
    ├─→ 计算技术指标：EMA(5,13,21) + MACD(5,13,5) + RSI(14) + BB(20,2.5) + ATR(14) + ADX(14) + OBV + VWAP + 成交量比
    │
    ├─→ 硬性条件判断
    │       ├─ 多头: RSI≤30 AND 价格≤BB下轨附近 AND ADX≥10
    │       └─ 空头: RSI≥70 AND 价格≥BB上轨附近 AND ADX≥10 AND (ADX≤45 OR ADX>50)
    │
    ├─→ 强趋势豁免检查 (ADX≥50 + |bb_pos|≥2.0)
    │       ├─ 是 → 跳过评分，直接做多/做空（评分×1.3）
    │       └─ 否 → 继续以下流程
    │
    ├─→ 加权评分卡（9项指标 × 权重） → 总分 -100 ~ +100
    │
    ├─→ 趋势强度修正
    │       ├─ ADX < 25 且总分绝对值 < 35 → 过滤为中性（观望）
    │       └─ ADX ≥ 35 → 总分 × 1.15
    │
    ├─→ 最终决策（动态阈值）
    │       ├─ ADX ≥ 35: 阈值降至 ±15
    │       ├─ 否则: 阈值 = ±30
    │       ├─ 总分 > 阈值  → 做多（LONG）
    │       ├─ 总分 < −阈值 → 做空（SHORT）
    │       └─ 否则 → 观望（NEUTRAL）
    │
    ├─→ 方向保护过滤
    │       ├─ ADX>45 且与EMA趋势相反 → 过滤（不做逆势）
    │       ├─ ADX>80 → 强制观望（极端趋势不开仓）
    │       └─ 通过 → 继续
    │
    ├─→ 冷却期检查
    │       ├─ 同方向间隔<2根K线 → 冷却跳过
    │       ├─ 但ADX>50+BB触边 → 豁免（间隔≥1根可进场）
    │       └─ 通过 → 执行交易
    │
    └─→ 计算出场位
            ├─ 止损: ATR × (1.2 ~ 2.0)
            └─ 止盈: ATR × (1.3 / 2.2 / 3.5 ~ 5.0)
```

## 九、2026-07-10 优化变更记录

| # | 变更内容 | 涉及文件 | 说明 |
|:-:|---------|---------|------|
| 1 | **强趋势豁免机制** | `__init__.py` `app.py` `engine.py` | ADX≥50 + `|bb_pos|≥2.0` 时绕过评分阈值，评分×1.3直接判定方向 |
| 2 | **动态评分阈值** | `__init__.py` | ADX≥35 时评分阈值从 ±30 降至 ±15，让更多强趋势信号通过 |
| 3 | **做空ADX限制放宽** | `app.py` `engine.py` | 做空ADX上界从 ≤45 改为 ≤45或>50，ADX>50+BB上轨时允许做空 |
| 4 | **冷却期豁免** | `app.py` | ADX>50 + BB触边时跳过冷却检查，但要求至少间隔1根K线（避免连续逆势） |
| 5 | **强趋势方向保护** | `app.py` `engine.py` | ADX>45 时只允许顺势交易（EMA方向），防止强下跌中抄底、强上涨中摸顶 |
| 6 | **极端趋势保护** | `app.py` `engine.py` | ADX>80 时强制观望，极端单边行情不开新仓 |
| 7 | **冷却期收紧** | `app.py` | cool_bars 从1改为2，正常信号至少间隔1根K线才允许同方向重复进场 |
| 8 | **RSI阈值调整** | `__init__.py` `app.py` `engine.py` | 做多RSI从≤20放宽至≤30，做空RSI从≥80放宽至≥70，信号量预计+30% |
